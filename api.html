Introduction

The Music AI API service allows developers to harness the power of artificial intelligence to generate and manipulate music. This documentation provides detailed information on API Endpoints, Request Parameters, Authentication and Response Formats.

Authentication

All API requests require authentication. To authenticate, include your API key in the authorization header as follows
Authorizaton: Bearer YOUR_API_KEY

Endpoints

1. Generate Melody

Generate a new melody using AI Algorithms

  • Endpoint: /melody
  • HTTP Method: POST

Parameters

  • Length (required): The length of the melody in bars or beats
  • Tempo (optional): The tempo of the melody in beats per minute (default: 120)
  • Key (optional): The musical key of the melody (default: C major)
  • Scale (optional): The musical scale of the melody (default: major)

Request Body

The request body should contain additional parameters or settings specific to the melody of the given algorithm

Response

The response will include a MIDI file representing the generated melody.

Example request

POST/melody

Authorizaton: Bearer YOUR_API_KEY

Remix Song

Remix an existing song using AI Algorithms

  • Endpoint: /remix
  • HTTP Method: POST

Parameters

  • song_id (required): The identifier of the song to remix
  • remix_type (optional): The type of remix to perform (e.g. "chill", "dance", "jazz").
  • remix_settings (optional): Additional settings or parameters specific to the remix algorithm.

    • Response

    The response will include an auido file representing the remixed song.

    Example request

    POST /remix

    Authorizaton: Bearer YOUR_API_KEY

    {

    "song_type" : "12345",

    "remix_type" : "dance",

    "remix_settings" :{

    "tempo" : 130

    "effects" : {

    "reverb" : true,

    "delay" : true

    }

    }

    HTTP/1.1 200 OK

    Content-type: audio/mp3

    remixed-audio.file

API Base URL

https://api.musicalservice.com/v1

Error Handling

In case of errors, the API will respond with appropriate status codes and error messages. Please refer to the Error Handling section for more details.
